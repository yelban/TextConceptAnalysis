このアーキテクチャについて

これは極めて洗練された高効率なプロンプト構造であり、「疑似コード（Pseudo-code）スタイル」と呼ばれています。 Geminiの保存機能（Saved info）の文字数制限に対応するため、「3段階分割コピー」機能を実装しました。

One-Click Install
ヒント: Geminiの「保存された情報」設定では、A → B → C の順に「+追加」しながら貼り付けてください。
ブロック A: コア論理 (Core) Role, Tone, Principles
ブロック B: モジュール上巻 (Modules 1-5) Visual, Research, Copy, Social, Slides
ブロック C: モジュール下巻 + 実行 (Modules 6-10) Writing, Sketch, Thought, Strategy, Tutor

コア分析：なぜこのプロンプトが有効なのか？

ROLE

Strategic AI Logic Engine

原理：役割の再定義。「アシスタント」や「友達」ではなく「エンジン」と定義することで、AIの語彙選択が変わり、雑談ではなく論理推論に特化します。

LANG

ja-JP(Tokyo) ONLY

原理：ローカライズの強制。ONLY と指定することで、不自然な翻訳調を排除し、東京のビジネスシーンで使われる自然な日本語を強制します。

CORE

Precision > Emotion

原理：重み付けの順序。「正確さを優先してください」という文章よりも、> 記号を使うほうがトークンを節約でき、かつ論理的な優先順位が明確になります。

[MODULES]

モジュール設計 / BLUF

原理：「思考の連鎖（Chain of Thought）」の応用です。4つのモードを事前定義することで、AIはタスクを分類するだけで適切な処理ロジックを呼び出せ、推論エラーを大幅に減らせます。

BLUF (Bottom Line Up Front)：軍事・ビジネス用語で「結論を先に」という意味。起承転結というAIの冗長な癖を強制的に矯正します。
[EXECUTION]

実行ループ

原理：ワークフローの定義。Output Essential Result Only -> Stop により、回答後の「お役に立てれば幸いです」といった無駄話を防止します。

Geminiでの設定方法 (推奨順)

方法 1：直接対話 (一時利用)

最速。一時的なモード切り替えやテストに最適。新しいチャットごとに貼り付けが必要。

  1. New Chat を開く
  2. チャット欄に完全なコードを貼り付け
  3. Enterキーで送信して適用確認
方法 2：Gemを作成 (上級者)

Gemini Advanced 適用。可建立多個分身，字數限制寬鬆。完全版 (Full)を推奨。

  1. Gem Managerを開く
  2. "New Gem" を選択
  3. Instructions に完全なコードを貼り付け
  4. 保存していつでも呼び出し可能に
方法 3：AI Studio (開発者)

最強。System Promptの重みを完全制御できる開発環境。プロンプトの微調整に最適。

  1. Google AI Studio へ移動
  2. System Instructions ブロックを見つける
  3. 完全なコードを貼り付け
  4. ロックしてテスト開始
👑 推奨 No.1
方法 4：保存された情報 (Saved Info)

設定は一度だけ。全ての新しい会話に自動適用されます。文字数制限が厳しいため分割版推奨。

  1. Gemini 設定 > 保存された情報へ
  2. Geminiへの指示 を見つける
  3. ブロック A を貼り付け
  4. + 追加 ボタンをクリック
  5. ブロック B を貼り付け
