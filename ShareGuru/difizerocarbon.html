核心方案

三大模組落地應用

思銳科技提供的不是一台主機，而是一條「具備資安圍籬的自動化生產力」

A

Coding Agent

讓 6 人團隊擁有 20 人產能

OpenCode GLM-4.7 / MiniMax M2.1 Anthropic Skills 技能框架

📋 場景一：碳盤查平台功能迭代

【導入前】

工程師接到需求「新增範疇三類別 4 的計算模組」，先花 3 天讀懂現有 codebase 的架構，再花 2 天寫程式，最後花 1 天 debug。一個小功能耗時一週。

【導入後】

Agent 自動掃 repo、產出架構圖、模組責任、關鍵資料流。工程師只需覆核 Agent 的設計方案，確認後 Agent 自動寫程式、跑測試、提交 PR。同樣功能從一週縮短到 1.5 天。

🧠 場景二：Skills 傳承垂直領域知識

舞雲的核心競爭力是什麼？是團隊累積的 ISO 14064 導入經驗、碳盤查計算邏輯、客戶產業特性判斷。這些知識過去只存在資深顧問腦中。

【Skills 方案】

將「ISO 14064 範疇判定規則」、「排放係數選用邏輯」、「盤查報告產出格式」封裝成 SKILL.md 技能包。新進顧問或 Agent 按需載入，立即具備專業判斷能力。知識不再只存在老員工腦中，而是成為公司的數位資產。

🌟 場景三：輔導客戶企業 AI 轉型導入（新商業模式）

這是舞雲智網最大的差異化機會。舞雲的客戶（台中精機、新呈工業等製造業）都面臨同樣的 AI 轉型焦慮：想用 AI 但不知道怎麼開始、擔心資料外洩、沒有專職 AI 團隊。

【Skills 顧問服務】

運用 Anthropic Skills 技能框架，協助客戶企業將其垂直領域知識（製程配方、品管標準、設備維護 SOP）封裝成可重複使用的技能包。

【商業價值】

舞雲從「碳盤查顧問」升級為「AI 轉型顧問」，開創新的收入來源。每家客戶的 Skills 導入專案可收取顧問費 NT$300,000-500,000。

B

數字人影音工廠

ESG 教育訓練影片的流水線

LTX-2 影片生成 XTTS v2 語音合成 Wav2Lip 嘴型同步 ComfyUI 工作流

🎬 場景：ESG 教育訓練影片系列化產出

舞雲需要為客戶製作一系列 ESG 教育訓練影片：「什麼是碳盤查」、「範疇一二三怎麼分」、「如何填寫盤查清冊」⋯⋯

【傳統做法】

外包影片製作，一支 60 秒影片報價 15 萬，10 支就是 150 萬。修改一個數據要重新配音、重新剪輯，額外收費。

【流水線做法】

腳本寫好丟進 LTX-2 流水線，數字人影片批次產出。同腳本可批次產出不同語言版本（中/英/日），修改數據只需改腳本重跑，成本是外包的 1/10。

C

企業知識庫 (RAG)

法規諮詢自動化

Qdrant / FAISS 向量資料庫 本地 Embedding 模型 RAG 檢索增強生成

📚 場景：ISO 14064 / ISO 50001 法規諮詢

【導入前】

客戶問「我們公司的柴油發電機算範疇一還是範疇二？」顧問翻 ISO 14064-1:2018 文件翻半天，找到答案後還要整理成客戶能懂的說法。同樣的問題下週另一個客戶又問一次。

【導入後】

RAG 知識庫一問就有：「根據 ISO 14064-1:2018 第 5.2.2 節，企業自有或控制的固定燃燒源（如柴油發電機）屬於類別 1（範疇一）直接排放。」答案附上「這段話出自哪份文件第幾頁」，顧問只需覆核正確性後轉發給客戶。

下一步建議

90 天落地藍圖

01

第 1-30 天：選定戰場

不要什麼都想做，先找最痛的那一個

「不要什麼都想做，先找最痛的那一個。」

  1. 1 盤點公司裡「花最多人力在抄、貼、對、找」的工作
  2. 2 從中選出 1 個最高頻、最痛、資料格式最固定的任務（建議優先選擇 RAG 知識庫）
  3. 3 定義「成功長什麼樣」——要有可量化的指標
  4. 4 準備 10-20 個真實案例作為測試資料
02

第 31-60 天：做出 MVP

先跑通一條線，再想怎麼擴張

  1. 1 部署本地推理服務，確認硬體正常
  2. 2 將 ISO 14064/50001 相關文件匯入 RAG 知識庫
  3. 3 建立「輸入 → 處理 → 驗證 → 輸出」四個步驟的檢查機制
  4. 4 用測試資料跑 10 次，記錄成功率與失敗原因
03

第 61-90 天：上線與迭代

讓它真的跑起來，然後收集回饋

  1. 1 找 2-3 位顧問做 beta 測試
  2. 2 建立「回報問題 → 修正 → 重測」的迴圈
  3. 3 記錄實際省下的時間 / 減少的錯誤 / 提升的產出
  4. 4 決定下一個要自動化的任務（Coding Agent 或數字人工廠）

不是買主機，是買一條生產線

很多人以為「買大主機就等於擁有 AI 能力」，這跟「買了健身器材就會變壯」的邏輯一樣天真。真正讓 AI 落地的關鍵在於：你有沒有把它變成一條可以重複運轉、可以被驗證對錯的生產線。
