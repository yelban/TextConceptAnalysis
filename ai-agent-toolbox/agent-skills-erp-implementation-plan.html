一、ERP vs Agent Skills 的定位

ERP 不只是「軟體」，而是三件事的組合

層面 ERP 提供的 Agent Skills 能做的
資料模型 標準化的會計科目、物料主檔、BOM 結構 ✗ 需要人為設計
業務邏輯 MRP 運算、成本分攤、稅務申報 △ 可實作，但需逐一開發
流程控制 審批流、權限、稽核軌跡 ✓ 可透過 Skills 實現
法規合規 符合 IFRS、稅務申報格式 ✗ 需要認證，AI 無法取代
生態整合 銀行、稅務機關、EDI 標準 △ 可串接，但需逐一處理

Agent Skills 真正的優勢

Agent Skills 不是要「取代」ERP，而是解決 ERP 的三大痛點：

痛點 1

輸入地獄

傳統：15 個欄位精確輸入

Agent：拍照 / 語音 / 自然語言

痛點 2

查詢困難

傳統：登入 → 找報表 → 讀數字

Agent：問一句話得到答案

痛點 3

例外處理

傳統：遇到非標準就卡住

Agent：提供選項、建議方案

二、七大應用場景

場景 1

採購入庫 — 從「輸入地獄」到「拍照搞定」

Skill: 智慧入庫助手
倉管：[拍照] 送貨單 Agent：我辨識到這是大成的送貨單 • 低筋麵粉 25kg × 20 包 = 500kg • 批號：L240115 • 效期：2025/01/15 • 建議倉位：A-03-02（同批號旁邊） 要我建立入庫單嗎？ 倉管：好，但放 A-03-01 Agent：已修改倉位為 A-03-01 入庫單 PO-2026-0115-001 已建立 請確認實際數量後點「完成入庫」
指標 傳統 ERP Agent 智慧層
每單輸入時間 3-5 分鐘 30 秒
錯誤率 5-8% < 1%
訓練時間 2 週 10 分鐘
場景 2

庫存查詢 — 從「開電腦查報表」到「問一句話」

Skill: 庫存問答助手
採購：低筋麵粉還有多少？夠不夠撐到下週？ Agent：目前低筋麵粉庫存狀況： 📦 總庫存：2,350 kg ├─ 台北廠：1,200 kg（批號 L240110，效期 01/10） ├─ 嘉義廠：1,150 kg（批號 L240115，效期 01/15） 📊 過去 7 天平均用量：380 kg/天 ⏱️ 預估可用天數：6.2 天 ⚠️ 下週有中秋備貨，預估用量會增加 40% 建議：本週四前補貨 1,000 kg 要我幫你建立採購申請嗎？
場景 3

訂單處理

LINE 自動接單，智慧判斷庫存

場景 4

AI 建議 + 人決策，自動通知異常

場景 5

場景 6

主動通知庫存、應收、品質異常
