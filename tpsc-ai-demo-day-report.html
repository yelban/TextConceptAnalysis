1

製程診斷自動化隊

工程單位

◆ 專案主題

AI 加速 Power Drop 真因尋找與改善 — 運用 DAO 框架結合 LLM 與代理工具（Manus、NotebookLM），分析太陽能製造中的複合性根因。

◆ 創新亮點

  • ✓ 數位人簡報：導入虛擬講師「小嚴」進行 24 小時全天候簡報
  • ✓ GPT 群聊功能：突破一對一模式，實現多人同時在線協作決策
  • ✓ 混血團隊：人類專家 + AI 工程師「小智」的深度結合

◆ 技術架構

Meeting Ink → GPT + Gemini → NotebookLM → Manus

◆ 關鍵效益

  • • 報告生成時間：4hr → 1hr（-75%）
  • • 釋放人力：0.5 FTE
  • • 試錯成本節省：20% NRE

🤖 AI 評審意見

✓ 優點

  • • 量化效益極為紮實，涵蓋 FTE 與 NRE 指標
  • • 將 AI 轉化為「知識資產」與「護城河」的戰略思維

△ 待改進

  • • Demo 證據較弱，多為靜態圖表而非操作流程
  • • 工具鏈整合鬆散，各工具間需手動觸發
2

資資不倦隊

人資單位

◆ 專案主題

智慧履歷篩選系統 — 利用 NotebookLM 建構履歷篩選與評分流程，透過 RAG 技術大幅縮減篩選時間。

◆ 系統功能

  • ✓ 精準提示詞工程：針對技術技能與必備條件進行權重比對
  • ✓ 批次處理：單次可處理 50 份履歷輸入
  • ✓ 雙版本輸出：簡報版 + 文字版結果報告

◆ 痛點量化

原痛點：單份履歷篩選 30 分鐘、單一職缺總耗時 4 小時+

◆ 效益數據

  • • 單份履歷：30min → 1.3min（-96%）
  • • 三份履歷合計僅需 4 分鐘
  • • 雙重提效：篩選 + 減少面試次數

🤖 AI 評審意見

✓ 優點

  • • 痛點與成效量化明確（30min → 1.3min）
  • • 未來規劃邏輯嚴密，含標準化模板與 SOP 設計

△ 待改進

  • • 依賴人工操作 NotebookLM，缺乏自動化串接
  • • 建議將節省時間換算為具體金額（NT$）或 FTE

AI Cloud 隊（舞雲智網）

資訊單位

◆ 專案主題

從個人工具到組織武器庫 — 利用 ChatGPT Project 功能結合 YAML 定義的任務鏈，將個人化 Prompt 轉化為標準化、可管理的組織資產。

◆ 四大痛點

語意失靈
品質失控
流程失序
管理失明

◆ 核心解法

ChatGPT 專案 + 任務鏈（Task Chain）

  • • 專案可分享給團隊成員
  • • 編輯指令封裝品牌風格
  • • YAML 定義標準執行步驟
  • • 可互動體驗連結

◆ 商業價值

協助 CLI 產出文章專案，獲得等值服務價值 NT$ 89,000

3-10倍
效率提升
50-70%
重工降低
1-3天
新人上手

🤖 AI 評審意見

✓ 優點

  • • 極佳的產品化思維：將 Prompt 工程透過 YAML 轉化為可管理的「任務鏈」資產
  • • 量化成果紮實：具備明確的 Before/After 對比與具體金額效益

△ 待改進

  • • 目前依賴 ChatGPT 介面手動操作，尚未達成完全自動化
  • • 高度依賴特定平台，若功能變動可能影響運作

總評：執行力極高且具備高度實用性的專案，成功解決企業導入 GenAI 時最頭痛的「標準化」問題，證明了 AI 可以從個人工具轉變為組織資產。

4

生產效率隊

生產單位

◆ 專案主題

Sputter 生產效率與機台健康監測 — 以功率數據規則化分析實現自動化 OEE 監測，成功預警真空泵異常。

◆ 分析維度

  • ✓ 曝光效率：全年 36 萬筆數據的波形分析
  • ✓ 切換成本：計算每次換線的時間成本
  • ✓ PM 效果：分析保養前後功率差異

◆ 痛點量化

原痛點：人工彙整 180 min/日、PM 前後差異難以量化

◆ 效益數據

  • • 節省 170 min/日 ≈ 56.7 hr/月
  • • 避免 72 hr 停機損失（122.4 cycles, 914.4 kWh）
  • • 回收 12 min/日產能

🤖 AI 評審意見

✓ 優點

  • • 量化效益極為紮實，涵蓋人力工時、電力能耗與產能損失
  • • 利用規則引擎解決 CSV 資料缺乏標籤的實務難題，展現極強落地執行力

△ 待改進

  • • Demo 形式較為靜態，多為分析圖表堆疊
  • • AI 用電模型 R² 僅 0.36，主要依賴規則運算

總評：極具實務價值的專案，透過精準的數據工程與規則分析，解決了高價值的製造業痛點。強烈建議投資推廣。

FA 貝氏優化隊

研發單位

◆ 專案主題

貝氏優化驅動智能開發流程 — 整合 BoTorch 貝氏優化與 Gemini LLM 建立雙引擎實驗設計平台，將鈣鈦礦材料開發流程標準化。

◆ 為何不用大型語言模型？

LLM 適合處理已知知識，但對於未知問題只能發散無法收斂。貝氏優化提供數學上絕對的精確度，可將問題收斂。

◆ 雙引擎架構

分析與建議新參數

L9 正交表
結合貝氏優化

🤖 AI 評審意見

✓ 優點

  • • 技術架構完整：結合貝氏優化的數學精確性與 LLM 的推理能力
  • • 具體的「實驗次數減少 60%」數據，有力證明方法論有效性

△ 待改進

  • • 實體實驗環節限制了全自動化的實現
  • • 建議將「減少 60%」換算成具體研發費用節省金額

總評：執行度極高且邏輯清晰的 AI 轉型專案，成功將 AI 引入傳統材料研發流程。若能補充具體節省金額，將具備極高商業價值。

設備玩家隊

  • ✓ 經驗孤島 → AI 統一解讀標準
  • ✓ 決策延遲 → T+0 即時預警

🤖 AI 評審意見

✓ 優點

  • • 痛點與效益量化非常具體（4小時縮減至2分鐘）
  • • DAO 架構清晰，用 Ragic 解決 AI 導入常見的 Data Garbage 問題

鈣鈦礦業務整合隊

業務行銷單位

◆ 專案主題

PSC 業務整合 AI 口袋工具 — 透過整合 AOI 瑕疵檢測與 Gemini Canvas 生成式 AI，建立業務「口袋小工具」。本次從「使用 AI」進化到「用 AI 創造工具」。

◆ 工具一：AOI 霧面檢測

  • • 智慧自動識別異常區域
  • • 可手動框選微小孔洞
  • • 自動計算寬度、高度、面積
  • • 產出異常報告供下載

◆ 工具二：AI 提案圖生成

  • • 選擇橫擺/直擺排列設計
  • • 風格選擇（科技感/高級感/寫實）
  • • 光影選擇（清晨/黃昏/陽光直射）
  • • 一次輸出四張不同角度圖片

◆ 技術實現

使用 Cursor 作為核心開發工具，透過概念描述自動生成程式碼，產出一頁式網頁工具

90%
一次性準確率
95%
時間減少（2hr→5min）

🤖 AI 評審意見

✓ 優點

  • • 痛點與解決方案對應精準，針對業務實際場景開發實用工具
  • • Demo 展示詳盡，介面友善且流程清晰，證明概念已落地為可用軟體

△ 待改進

  • • 缺乏全自動化流程設計，目前仍依賴人工單次操作
  • • ROI 僅停留在單次任務的時間節省比例，缺乏總體財務影響換算

總評：務實且完成度高的業務賦能專案。雖然技術上未達全自動化 Pipeline，但作為輔助工具已具備極高的實用性與落地價值。建議推動其 API 化以擴大影響力。
