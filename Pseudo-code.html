關於此架構

這是一個非常精煉且高效率的 Prompt 架構，被稱為「偽代碼（Pseudo-code）風格」。 針對 Gemini 網頁版嚴格的字數限制，本工具採用「三段式拆分」策略，確保所有指令都能完整植入。

One-Click Install
操作提示：在 Gemini「給 Gemini 的指令」中，依序貼上 A → 點擊「+ 新增」→ 貼上 B → 點擊「+ 新增」→ 貼上 C。
區塊 A: 核心邏輯 (Core) Role, Tone, Principles
區塊 B: 模組上卷 (Modules 1-5) Visual, Research, Copy, Social, Slides
區塊 C: 模組下卷 + 執行 (Modules 6-10) Writing, Sketch, Thought, Strategy, Tutor

核心解析：為什麼這段 Prompt 有效？

ROLE

Strategic AI Logic Engine

原理：重新定義角色。不再是「助手」或「朋友」，而是「引擎」。這會切換模型的用詞庫，使其更傾向於邏輯推演而非社交閒聊。

LANG

zh-TW(Taiwan) ONLY

原理：強制本地化。特別強調 ONLY 和 local terms，是為了對抗模型訓練資料中大量的簡體中文權重，確保輸出是「影片」而不是「視頻」、「品質」而不是「質量」。

CORE

Precision > Emotion

原理：權重排序。使用 > 符號比文字說明「請優先考慮精確度」更節省 Token，且邏輯層級更明確。

[MODULES]

模組化設計 / BLUF

原理：這是「思維鏈（Chain of Thought）」的變體。預先定義四種模式，讓模型在接收任務時，只需做「分類」動作，就能調用對應的處理邏輯，大幅降低推理錯誤率。

BLUF (Bottom Line Up Front)：在文案模式中，這是一個軍事/商業溝通術語，意指「結論先行」，強迫模型改變「起承轉合」的囉唆習慣。
[EXECUTION]

執行閉環

原理：定義工作流。Output Essential Result Only -> Stop 是為了防止模型在回答完後，又自動生成「希望這對您有幫助」這類的廢話。

如何在 Gemini 中設置？ (4種方式)

方式 1：直接對話 (臨時用)

最快速，適用於臨時需要切換模式時。

  1. 開啟 New Chat
  2. 在對話框貼上完整代碼
  3. 按 Enter 發送，確認回應後開始任務
方式 2：建立 Gem (進階版)

Gemini Advanced 適用。可建立多個分身，字數限制寬鬆。完全版 (Full)を推奨。

  1. 點擊 Gem Manager > "New Gem"
  2. 在 Instructions 欄位貼上完整代碼
  3. 儲存後可隨時切換調用
方式 3：AI Studio (開發者)

最強大，完全控制 System Prompt 權重，適合測試。

  1. 前往 Google AI Studio
  2. 找到 System Instructions 區塊
  3. 貼上完整代碼
  4. 鎖定後開始測試
👑 推薦首選
方式 4：個人化指令 (Saved Info)

設定一次，所有新對話自動套用。文字上限較嚴格，請用拆分版。

  1. 進入 Gemini 設定 > 個人化建議
  2. 找到 給 Gemini 的指令
  3. 貼上 區塊 A
  4. 點擊 + 新增 (重複 2 次)
  5. 分別貼上 區塊 B 與 區塊 C
