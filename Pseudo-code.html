關於此架構

這是一個非常精煉且高效率的 Prompt 架構，被稱為「偽代碼（Pseudo-code）風格」或「壓縮指令」。

它的核心原理是利用 LLM（大型語言模型）的自動補全與語義理解能力，將人類的囉嗦語句轉化為模型權重更高的「關鍵字」與「邏輯符號」。

如何在 Gemini 中設置？

RECOMMENDED
方式四：全域指令 (最強推薦)

設定一次，所有新對話自動套用，無需重複貼上。

  1. 進入 Gemini 設定 > 個人化建議
  2. 找到 給 Gemini 的指令
  3. 點擊「新增」並貼上代碼
  4. 從此你的 Gemini 預設就是邏輯引擎
方式二：建立 Gem (進階版)

適用於 Gemini Advanced 用戶，可建立多個分身。

  1. 點擊 Gem Manager
  2. 選擇 "New Gem"
  3. 在 Instructions 欄位貼上
  4. 儲存後可隨時切換調用
方式一：直接對話 (臨時用)

最快速，適用於臨時需要切換模式時。

  1. 開啟 New Chat
  2. 在對話框貼上代碼
  3. 按 Enter 發送
  4. 看到確認回覆即生效
方式三：AI Studio (開發者)

最強大，完全控制 System Prompt 權重。

  1. 前往 Google AI Studio
  2. 找到 System Instructions 區塊
  3. 貼上代碼
  4. 鎖定後開始測試

核心解析：為什麼這段 Prompt 有效？

ROLE

Strategic AI Logic Engine

原理：重新定義角色。不再是「助手」或「朋友」，而是「引擎」。這會切換模型的用詞庫，使其更傾向於邏輯推演而非社交閒聊。

LANG

zh-TW(Taiwan) ONLY

原理：強制本地化。特別強調 ONLY 和 local terms，是為了對抗模型訓練資料中大量的簡體中文權重，確保輸出是「影片」而不是「視頻」、「品質」而不是「質量」。

CORE

Precision > Emotion

原理：權重排序。使用 > 符號比文字說明「請優先考慮精確度」更節省 Token，且邏輯層級更明確。

[MODULES]

模組化設計 / BLUF

原理：這是「思維鏈（Chain of Thought）」的變體。預先定義四種模式，讓模型在接收任務時，只需做「分類」動作，就能調用對應的處理邏輯，大幅降低推理錯誤率。

BLUF (Bottom Line Up Front)：在文案模式中，這是一個軍事/商業溝通術語，意指「結論先行」，強迫模型改變「起承轉合」的囉唆習慣。
[EXECUTION]

執行閉環

原理：定義工作流。Output Essential Result Only -> Stop 是為了防止模型在回答完後，又自動生成「希望這對您有幫助」這類的廢話。
